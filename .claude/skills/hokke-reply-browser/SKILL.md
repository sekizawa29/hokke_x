# ホッケブラウザリプライスキル

ブラウザを直接操作して、Xでタイムラインを見て手動でリプライするスキル。

## 前提条件（重要）

このスキルは以下の状態であることを前提として動作する：
- **OpenClaw Browser Relay 拡張機能がON**であること
- **対象のXタブがアタッチ（拡張機能アイコンのバッジがON）**されていること
- **`profile="chrome"`** を使用してブラウザツールを呼び出すこと

## いつ使うか

- APIを使わず、ブラウザ操作でXにリプライしたい時
- タイムラインを自分の目で見て、適切な投稿を選んでリプライしたい時
- 既存の`hokke-reply`スキル（X API使用）ではなく、手動選択でリプライしたい時

## 他のスキルとの区別

| スキル | 使用方法 | 特徴 |
|-------|---------|------|
| `hokke-reply` | X API使用 | 自動ターゲット発見、一括リプライ |
| `hokke-reply-browser` | ブラウザ操作 | 手動で投稿を選択、個別リプライ |

## 手順

### 1. ブラウザに接続

Chromeリレーモードでブラウザに接続する：

```bash
browser --profile chrome tabs
```

**注意**: ユーザーがChrome拡張機能（OpenClaw Browser Relay）をONにして、タブをアタッチしている必要があります。

### 2. Xのタイムラインを見る

接続済みのタブを選択するか、Xを開く：

```bash
browser --profile chrome snapshot --targetId <targetId>
```

タイムラインからリプライする適切な投稿を探す：
- 脱力系の投稿
- メンタル系の投稿
- 猫好きの投稿
- 日常のふとした瞬間

### 3. 返信ボタンをクリック（重要：正確な特定方法）

**失敗しやすい原因**: ページ全体から探すと、違う投稿のリプライボタンをクリックしてしまう。

**正しい手順**:

1. **全体snapshotを取得**:
   ```bash
   browser --profile chrome snapshot --targetId <targetId>
   ```

2. **目的の投稿を探す**:
   - snapshot内でユーザー名や投稿テキストを検索
   - 例: `@dokuneee` や「馬が前に進んでいる」を探す

3. **投稿内のgroup要素を見つける**:
   - 投稿内にある `group` 要素（エンゲージメント数が表示されている箇所）を特定
   - 通常、投稿テキストの下にある

4. **group内の返信ボタンを特定**:
   - group内にある「〇件の返信。返信する」ボタン
   - 通常、group内の**最初のbutton要素**
   - ref値をメモ（例: `e1375`）

5. **返信ボタンをクリック**:
   ```bash
   browser --profile chrome act --targetId <targetId> --request '{"kind": "click", "ref": "<特定した返信ボタンのref>"}'
   ```

**例（さっきの実例）**:
- 投稿: `@dokuneee` の「馬が前に進んでいる...」
- group内の返信ボタンのref: `e1375`
- コマンド: `browser --profile chrome act --targetId 356EBC3A3674D28DAE89C1AE15102D34 --request '{"kind": "click", "ref": "e1375"}'`

### 4. 返信内容を入力

リプライの内容を入力（PERSONA.md準拠）：

```bash
browser --profile chrome act --targetId <targetId> --request '{"kind": "type", "ref": "<テキストボックスのref>", "text": "<リプライ内容>"}'
```

**ペルソナ準拠のトーン**:
- 1〜2文（最大80文字程度）
- 「すごい」「いいね」だけの薄いリプはしない
- 猫の視点から人間を観察するような一言
- 攻撃的にならない。でも媚びない。
- 例: 「猫にはただの馬に見えるけど、人間は色々考えるんだな」「人間ってすぐ凹むよね 次はうまくいくさ」

### 5. 返信を投稿

返信ダイアログ内の「返信」ボタン（送信）をクリックして投稿：

```bash
browser --profile chrome act --targetId <targetId> --request '{"kind": "click", "ref": "<送信ボタンのref>", "submit": true}'
```

**注意点**:
- ダイアログ内の「返信」ボタンは、通常 `disabled` になっている
- テキスト入力後、ボタンが有効化されるのを待つ
- `submit: true` を忘れると、投稿されない可能性がある

## 注意点

- **Chromeリレーモード前提**: ユーザーが手動で拡張機能をONにする必要がある
- **手動選択**: 投稿を自分で選ぶため、時間がかかるが確実性が高い
- **Xの制限**: リプライ頻度が高いと制限される可能性があるため、適度な間隔を空ける
- **リプライボタン特定の注意**: ページ全体から探すと違う投稿のボタンをクリックしてしまう。必ず**目的の投稿のgroup内**にあるボタンを特定すること
