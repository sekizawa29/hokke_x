# hokke-claude-consult

`claude -p` を使ってClaude Codeへ相談し、回答を運用に反映するためのスキル。

## 使うタイミング

- 投稿/リプライ運用の方針を比較したい時
- KPIや頻度設計の妥当性を第三者視点で確認したい時
- 実装前に運用案を壁打ちしたい時

## 実行

```bash
cd ~/pjt/hokke_x
bash .claude/skills/hokke-claude-consult/scripts/ask.sh "相談内容"
```

直接実行する場合:

```bash
claude -p "相談内容"
```

## 相談文テンプレート

以下の3要素を入れる:

1. 前提: アカウント状況・制約（例: 投稿4-5件/日、炎上回避）
2. 目的: 何を最大化するか（例: フォロワー増、プロフィール遷移）
3. 出力形式: 欲しい粒度（例: 件数/時間帯/KPI/調整ルール）

例:

```text
ホッケX運用。投稿4-5件/日。リプライを定期実行で成長させたい。
スパム感を抑えつつフォロワー増を狙う最適案を提案して。
前提: 日本語アカウント、猫ペルソナ、政治炎上は回避。
出力: 件数、時間帯、頻度、2週間後の調整ルールを具体化。
```

## 出力の扱い

回答をそのまま採用せず、以下で整理して返す:

- 提案要約（3-5行）
- 採用する設定値
- 見送る案と理由
- 次の実装タスク

## 注意

- `claude -p` は環境によって `EACCES` やネットワークエラーが起きることがある
- エラー時は原因（権限/接続）を明示し、ローカルの暫定案を提示して進める
